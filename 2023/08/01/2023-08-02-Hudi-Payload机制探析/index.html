Hudi Payload机制探析

发表于 2023-08-02 更新于 2023-08-03 分类于 大数据

Apache Hudi的Payload是一种可扩展的数据处理机制，通过不同的 Payload 我们可以实现复杂场景的定制化数据写入方式，大大增加了数据处理的灵活性。Hudi Payload在写入和读取Hudi表时对数据进行去重、过滤、合并等操作。本文我们会深入探讨Hudi Payload的机制和不同Payload的区别及使用场景。

作用机制

Hudi在预聚合阶段，根据RecordPayload的实现执行数据去重。对于COW表通过RecordPayload合并传入记录与存储上现有记录。对于MOR表读取及合并，也会根据RecordPayload实现执行DeltaLog和BaseParquet的合并。

参数名 默认值 备注
hoodie.datasource.write.payload.class org.apache.hudi.common.model.OverwriteWithLatestAvroPayload (Optional) MOR和COW写入时作用
hoodie.datasource.compact.payload.class org.apache.hudi.common.model.OverwriteWithLatestAvroPayload (Optional) MOR合并和读取时作用

这里需要强调的是：COW表的数据预合并和合并均在写入过程完成，逻辑较为清晰，在preCombine和tagging阶段即可生成数据的最终结果。而MOR表是将Commit的数据原样的保存到DeltaLog中，即在DeltaLog中记录了每一次Commit的真实记录，在合并和读取阶段才对DeltaLog中的数据进行preCombine后再与BaseFile中的数据进行合并，当然在写入阶段其也执行了数据预合并以较少数据的写入量。对于MOR表，需要注意compact和write时payload参数不同时的影响。

下图列举了HoodieRecordPayload接口需要实现的方法，这里有两个重要的方法preCombine和combineAndGetUpdateValue，下面我们对这两个方法进行分析。

  • preCombine：定义了两个payload合并的逻辑。该方法会在数据入湖前对数据进行去重。对于MOR表，会对DeltaLog里的数据进行合并。

  • combineAndGetUpdateValue：该方法将现有湖里的BaseFile中的数据与新数据进行对比合并，返回需要持久化的新数据。对于MOR表，会在读取及合并时使用该方法将BaseFile和经过preCombine合并后的数据进行进一步合并。

  • getInsertValue：该方法定义了如何将数据从payload转换为GenericRecord，以便于真实入湖。

常用实现

Hudi所有Payload都实现HoodieRecordPayload接口，下面列出了所有实现该接口的预置常用Payload类：

实现类 含义
OverwriteWithLatestAvroPayload preCombine选取order字段最新的记录，combineAndGetUpdateValue采用正在写入的记录。
DefaultHoodieRecordPayload preCombine和combineAndGetUpdateValue均选取order字段最新的记录。
ExpressionPayload preCombine选取order字段最新的记录，combineAndGetUpdateValue基于SQL中的表达式逻辑。
OverwriteNonDefaultsWithLatestAvroPayload preCombine选取order字段最新的记录，combineAndGetUpdateValue覆盖存储中非默认值字段。
PartialUpdateAvroPayload preCombine选取非NULL字段，combineAndGetUpdateValue覆盖存储中非NULL字段。

还有很多其他实现，例MySqlDebeziumAvroPayload 和 PostgresDebeziumAvroPayload 支持无缝应用通过 Debezium for MySQL 和 PostgresDB 捕获的更改。

使用案例

  • SparkSQL的MergeInto：使用ExpressionPayload，其在preCombine会取preCombineField字段值较大的数据，在combineAndGetUpdateValue阶段会更根据SQL中的表达式逻辑进行合并。需要注意的是当使用MOR表时，update不支持选择target中的字段，且所有字段都必须赋值，主要是因为MOR表在MergeInto写入阶段不和BaseFile中的数据进行合并，仅将数据写入DeltaLog中，合并表达式无效。

  • 字段值分批到达场景：使用PartialUpdateAvroPayload，对于未到达字段默认赋值为NULL，在后续字段到达时，更新NULL字段，已达到数据合并效果。该策略在COW表和MOR表场景下均适用。
